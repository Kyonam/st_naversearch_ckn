# 네이버 오픈API 공통 가이드 (README)

네이버 오픈API를 사용하여 클라이언트 애플리케이션을 개발하기 위해 반드시 숙지해야 할 공통 가이드입니다.

## 1. 서론
네이버 오픈API는 네이버가 보유한 핵심 기능(검색, 지도, 데이터랩, 클로바 등)을 외부 개발자들이 자신의 서비스에 활용할 수 있도록 제공하는 인터페이스입니다.

## 2. API 인증 방식 분류
네이버 API는 크게 두 가지 인증 방식으로 나뉩니다.

### 2.1. 비로그인 방식 (HTTP Header 방식)
- 사용자가 네이버 계정으로 로그인할 필요가 없는 API입니다.
- **인증 수단**: 클라이언트 아이디, 클라이언트 시크릿
- **해당 API**: 검색, 데이터랩, 캡차, 공유하기, 단축 URL 등.

### 2.2. 로그인 방식 (OAuth 2.0 방식)
- 사용자의 권한이 필요한 API입니다.
- **인증 수단**: 액세스 토큰 (Access Token, OAuth 2.0 기반)
- **해당 API**: 네이버 로그인, 내 정보 조회, 카페 게시글 쓰기, 캘린더 일정 관리 등.

## 3. 개발 절차 전체 흐름
1. **애플리케이션 등록**: [네이버 개발자 센터]에서 앱 등록 후 Client ID/Secret 발급.
2. **API 권한 설정**: 사용할 API 기능을 활성화.
3. **인증 코드 구현**: 비로그인 방식은 헤더 추가, 로그인 방식은 OAuth 2.0 연동.
4. **API 호출**: RESTful 방식으로 엔드포인트에 요청 전송.
5. **응답 처리**: JSON 또는 XML 형식의 데이터 파싱 및 UI 반영.

## 4. 공통 오류 처리
네이버 API는 표준 HTTP 응답 코드를 사용하여 오류를 나타냅니다.
- **200 OK**: 요청 성공.
- **400 Bad Request**: 요청 파라미터가 잘못된 경우. (대문자/소문자 구분 주의)
- **401 Unauthorized**: 인증에 실패한 경우. (Secret 오타 또는 앱 삭제 등)
- **403 Forbidden**: API 권한 설정이 안 되어 있거나 하루 할당량을 초과한 경우.
- **429 Too Many Requests**: 초당 호출 제한을 넘긴 경우.
- **500 Internal Server Error**: 네이버 서버 내부 오류.
